# 戸籍研究月例会 — 運用テンプレ

アプリ実装前の **運用だけ** を揃えるためのフォルダです。  
正本は **Git の Markdown（1か月＝1ファイル）**、資料原本は **クラウド（Google ドライブ等）** です。

## 役割分担

| 置き場 | 中身 |
|--------|------|
| このリポジトリ `content/koseki-monthly-meeting/*.md` | 開催記録（事務連絡・協議・研修など）。`status` で下書き／公開を区別 |
| クラウド `月例会アーカイブ/` | PDF・画像などの原本。リポジトリには載せない |

## Google ドライブ側のフォルダ構成

トップに **1つだけ** 正本フォルダを置き、月ごとに分けます。

```
月例会アーカイブ/
└── YYYY-MM_短いテーマ/          … 例: 2026-05_除籍の読み方
    └── 01_資料/                 … 配布PDF・スキャン等（ファイル名は原本のまま）
```

- フォルダ名は **`YYYY-MM_短いテーマ`** に統一（回数ではなくテーマ基準）
- 同名ファイルがあるときは末尾に `_氏名` または `_v2`
- 録画・会場写真は置かない（概要メモと事前共有資料のみ）

## 見出し構成（現行）

各月次ファイルの本文は次の順序で書きます。

| 見出し | 内容 |
|--------|------|
| 事務連絡 | 次回予定・宿題・出欠など |
| 協議 | 当日の論点・未解決事項 |
| 研修 | 研修内容（なければ「なし」） |
| その他 | 上記に当てはまらないメモ |
| 資料 | `[表示名](https://...)` 形式のリンク |

旧テンプレ（論点・資料リンク・未解決・次回予定・その他メモ）で書かれたファイルも、読み込み時に上記へ自動変換されます。

## 書記の流れ

1. **開催日当日** — `_template.md` を複製し、`status: draft` のまま必須項目を埋める
2. **1週間以内** — 清書し `status: published` に変更（書記＋運営コアが確認）
3. **資料** — `01_資料/` にアップロードし、Markdown の「資料」に URL を貼る

### 忙しい月（最小限）

次の4点だけでも可（研修・その他は「なし」でよい）。

- 開催日
- 協議（箇条書き）
- 資料（リンク）
- 事務連絡（次回予定・宿題）

## ファイル一覧

| ファイル | 用途 |
|----------|------|
| `_template.md` | 新規月次のひな形（複製して使う） |
| `2026-05_サンプルテーマ.md` | 記入例（`published`） |
| `drive-stub/` | ドライブに作るフォルダ構成の見本（中身は `.gitkeep` のみ） |

## Web アプリ（MVP）

| URL | 用途 |
|-----|------|
| `/koseki` | 公開済み（`published`）の一覧・詳細 |
| `/koseki/edit` | 編集（要 `KOSEKI_EDIT_PASSWORD`） |

`.env.local` に `KOSEKI_EDIT_PASSWORD` を設定してから `npm run dev` で起動してください。

- `draft` は編集画面のみ。`published` だけ会員向け一覧に表示
- 資料リンクは `http://` / `https://` のみ保存・表示可能
- Neon（PostgreSQL）を使う場合は `DATABASE_URL` を設定し、`npm run db:setup` で表作成・Markdown取り込み
- カレンダー連携・コメント機能は MVP 対象外
